## Vision

### 10x Check
The 10x version of Tensogram isn't just a wire format library. It's ECMWF's internal data plane, replacing GRIB, NetCDF, BUFR, and ODB for all internal data movement. Every pipeline component speaks Tensogram. FDB stores Tensogram natively. Zarr v3 uses Tensogram as a backend store. The CLI becomes the swiss army knife that ecCodes' grib_* tools are today. ML training pipelines read Tensogram directly via PyTorch/NumPy integration. Partial range decode means FDB can serve sub-field extractions without reading full messages. The encoding pipeline evolves to include ZFP and SZ3 for error-bounded lossy compression of ML model outputs.

## Implementation Decisions
- **CBOR crate:** ciborium (built-in canonical encoding, serde integration)
- **Approach:** Full Stack (all 5 crates + accepted expansions from day 1)
- **Feature gates:** `async` for tokio support, `mmap` for memory-mapped file access
- **Encode strategy:** Seekable buffer with placeholder backfill (v1). Streaming mode (total_length=0) for non-seekable outputs.
- **NaN handling:** simple_packing rejects NaN inputs with EncodingError
- **Zero-object messages:** Valid (metadata-only messages for pipeline signaling)
- **Payload order:** Always C-order (row-major). Strides describe logical interpretation after decode.
- **Shuffle + partial range decode:** Don't compose. Document that partial range only works for simple_packing+szip and uncompressed.
- **`tensogram set` safety:** Distinguish mutable keys (MARS, units, scientific) from immutable keys (shape, strides, dtype, encoding, hash). Reject immutable key modifications.
- **Parallel arrays:** objects[] and payload[] remain separate (allows skipping payload decode for metadata listing). Validate objects.len == payload.len.
- **Async + compression:** Use spawn_blocking for libaec FFI calls in async context.
- **Binary header index:** Object byte offsets live in a fixed-size binary header prefix (num_data_objects + object_offsets[]), not in CBOR. Eliminates CBOR circular dependency during encoding. metadata_offset field accommodates optional padding before CBOR block.
- **libaec partial range:** libaec has no aec_decode_range(). Implement RSI block seeking manually (~200-300 LOC in partial_decode.rs), porting GribJump's Aec.h/Ccsds.h approach.
- **C FFI:** Opaque handle pattern. Typed getters with dot-notation key paths (e.g., tgm_metadata_get_string(meta, "mars.param", &val)). cbindgen generates headers.
