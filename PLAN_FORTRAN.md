# Tensogram Fortran Interface — Development Plan

> **Status.** The **synchronous surface is implemented** on branch
> `feat/fortran-interface` (PR #127): generic encode/decode (dtype × rank),
> the multi-message file API, application metadata + the encoding pipeline,
> and the streaming encoder — with bidirectional cross-language parity
> (C/C++ and Python), an error-enum↔header consistency guard, and a CI
> lane. Only the **async surface is deferred** (§7). This document remains
> the durable design + roadmap; per-milestone status is marked in §7 and
> the backward-looking record of merged work is `CHANGELOG.md`.
> It is grounded in a direct reading of the `tensogram-ffi` C ABI at
> version **0.21.0**.

---

## 1. Goal and scope

Provide a **shallow, idiomatic Fortran interface** that lets scientific
Fortran codes (ECMWF's IFS and similar HPC/NWP codes) **read and write
Tensogram messages** directly — encode native Fortran arrays into `.tgm`
bytes/files and decode `.tgm` back into native Fortran arrays — without
dropping to C by hand.

In scope (in priority order):

1. Synchronous **encode / decode** of N-dimensional numeric arrays.
2. The **file API** (`.tgm` multi-message: open, append, random-access
   decode, iterate) — the common NWP "append forecast steps" pattern.
3. **Metadata** marshalling (descriptors + vocabulary-agnostic
   application metadata) across the boundary.
4. Native `real(:,:)` (and other rank/dtype) **array ergonomics** so the
   user never hand-rolls `c_loc`, strides, or byte counts.

Deferred (the streaming encoder was originally listed here but has since
shipped — see §7):

- The entire **async** surface (it uses C completion callbacks — the
  hardest part to bind and the least in demand for NWP codes, which are
  happy with blocking calls).
- `CFI_cdesc_t` / TS 29113 descriptor passing (only needed for
  zero-copy *non-contiguous* input — see §5.2).

Non-goals: re-implementing any tensor/codec logic in Fortran (everything
delegates to `libtensogram`); interpreting metadata vocabularies (the
library is vocabulary-agnostic and so is this binding).

## 2. Why this is low-risk: the FFI was built for exactly this

The project's own roadmap already listed *"Fortran interface over Rust"*
and rated it **"Low complexity — ISO_C_BINDING over existing C FFI."**
That assessment holds up against the actual code:

- There is a **purpose-built, flat C ABI** in the `tensogram-ffi` crate
  (`rust/tensogram-ffi/tensogram.h`, generated by `cbindgen`), already
  consumed by the header-only C++ wrapper. A Fortran binding is the same
  move the C++ wrapper makes, expressed in Fortran.
- The ABI is **C, not C++** — no name mangling, no exceptions, no
  vtables. Everything Fortran's `iso_c_binding` needs is already
  exposed.
- Distribution is **already mature**: `cargo-c` produces a versioned
  SONAME, a `tensogram.pc` pkg-config file, and an installed
  `tensogram/tensogram.h`; tagged releases ship prebuilt tarballs. The
  Fortran binding links via `pkg-config --cflags --libs tensogram` like
  any other consumer.

We are doing the thing the architecture was designed to allow, not
breaking new ground.

## 3. Verified facts about the C ABI (the foundation)

Read directly from `rust/tensogram-ffi/tensogram.h` and
`docs/src/guide/c-api.md` at 0.21.0. These are the load-bearing facts a
Fortran binding sits on:

- **Flat, `tgm_`-prefixed functions**; ~90 of them spanning sync
  encode/decode, the file API, scan/iterators, validation, streaming,
  and async.
- **Opaque handles** for all state: `tgm_message_t*`, `tgm_file_t*`,
  `tgm_metadata_t*`, `tgm_streaming_encoder_t*`, iterators, etc. Each has
  an explicit `*_free` / `*_close`. Callers never see the inner structs.
- **Only a few non-opaque POD structs** to mirror:
  - `tgm_bytes_t { uint8_t *data; size_t len; }` — owned byte buffer
    returned by encode; freed with `tgm_bytes_free` (**passed by
    value**, easy to get wrong).
  - `tgm_scan_entry_t { size_t offset; size_t length; }`.
  - `TgmEncodeMaskOptions` / `TgmDecodeMaskOptions` (only needed for the
    NaN/Inf opt-in paths — not required for the first arc).
- **Metadata crosses the boundary as a JSON string**
  (`const char *metadata_json`), *not* as nested CBOR/struct types. The
  Fortran layer only ever marshals **text**, never models the
  vocabulary. The JSON schema is flat:
  `{"descriptors":[ {…per-object…} ], "base":[ {…app metadata…} ], …}`;
  unknown top-level keys flow into `_extra_`.
- **Tensor data crosses as raw bytes**: `tgm_encode` takes
  `data_ptrs` (array of `const uint8_t*`) + `data_lens`; shape / dtype /
  strides live in the JSON. **This is the fact that makes native
  `real(:,:)` possible with zero C/Rust changes (§5.2).**
- **Errors are a C enum returned by value** plus a thread-local string:
  - `tgm_error` — `TGM_ERROR_OK = 0`, `TGM_ERROR_FRAMING = 1`,
    `TGM_ERROR_METADATA = 2`, `TGM_ERROR_ENCODING = 3`,
    `TGM_ERROR_COMPRESSION = 4`, `TGM_ERROR_OBJECT = 5`,
    `TGM_ERROR_IO = 6`, `TGM_ERROR_HASH_MISMATCH = 7`,
    `TGM_ERROR_INVALID_ARG = 8`, `TGM_ERROR_END_OF_ITER = 9`,
    `TGM_ERROR_REMOTE = 10`, `TGM_ERROR_MISSING_HASH = 11`,
    `TGM_ERROR_TIMEOUT = 12`, `TGM_ERROR_CANCELLED = 13`.
  - `const char *tgm_last_error(void)` — thread-local, valid until the
    next `tgm_*` call on the thread.
  - `const char *tgm_error_string(tgm_error)` — static description.
  - The enum is **non-exhaustive** by policy; treat any non-`OK` as
    failure.

## 4. Chosen approach: Path A + native `CONTIGUOUS` arrays

**Path A — a hand-written `iso_c_binding` Fortran module library.** A
`fortran/src/tensogram.f90` module with one private `interface` block
declaring each needed `tgm_*` function as `bind(C, name="tgm_…")`, plus a
public idiomatic layer on top that handles string marshalling, the
`tgm_bytes_t` round-trip, error→`ierr` mapping, derived types wrapping
the opaque handles with `final` procedures, and the column-major
contract. This is exactly what `eccodes_f90` does and what IFS users
already expect to link against.

The binding **accepts native `real(:,:)`** (and other ranks/dtypes)
**entirely in the Fortran layer, with zero changes to the C or Rust
code** — see §5.2 for why this is possible.

### 4.1 Alternatives considered, and why they're rejected or deferred

| Alternative | Verdict | Why |
|---|---|---|
| **Path B — auto-generate the raw `bind(C)` layer** (SWIG-Fortran, or a small generator in `tensogram-ffi/build.rs` beside cbindgen) | **Optional accelerator, not a replacement** | The header is itself machine-generated, so mechanising the 1:1 layer fits the project's "derived artefacts have one source of truth" rule. But *no* generator removes the hand-written ergonomic layer (strings, arrays, lifetimes), and SWIG-Fortran is far less battle-tested than its C++/Python backends. Treat as a way to reduce boilerplate churn against the pre-1.0 ABI — not a different deliverable. |
| **Path C — Fortran-native C entry points in Rust** (`CFI_cdesc_t` / TS 29113, a `tensogram-fortran` shim crate) | **Deferred** | Genuinely native feel (pass assumed-shape arrays, handle column-major in Rust), but couples a language-specific concern into the language-neutral FFI, and its only real payoff — zero-copy *non-contiguous* input — is marginal for a serializer (the wire format is contiguous bytes; a non-contiguous array must be gathered regardless, which Fortran already does for free). Revisit only if a concrete zero-copy-non-contiguous need appears. |
| **Bind through the C++ wrapper** (`cpp/include/tensogram.hpp`) | **Rejected** | Fortran↔C++ needs C shims anyway; the C++ layer adds nothing and subtracts ABI cleanliness. |
| **Ship no library, document `interface` blocks only** | **Rejected (except as a throwaway PoC)** | Gives users the raw C-style API with hand-built JSON and manual `c_loc`/free — no `real(:,:)`, no finalizers, no error mapping. |

## 5. The decisions that dominate the effort

The path choice is the easy part. These are where the work and the risk
actually live.

### 5.1 Memory order — the column-major contract (THE decision)

Tensogram stores `shape` **and** `strides` explicitly and is
layout-agnostic. `docs/src/concepts/objects.md` states it plainly: a
C-contiguous `[100,200]` array has strides `[200,1]`; *"for a
Fortran-contiguous (column-major) array the strides would be reversed:
`[1, 100]`."*

Three possible contracts:

1. **Pass verbatim** — make the user supply column-major shape+strides.
   Leaks layout into every call site. Rejected.
2. **Reverse shape+strides at the boundary** — a Fortran `a(ni,nj)` is
   written with on-wire descriptor shape `[nj, ni]` and C-contiguous
   strides `[ni, 1]`. The bytes are already column-major (`i` fastest),
   so this is a pure metadata relabelling, **zero data movement**.
3. **Physically transpose** so a Python/C reader sees the same *logical*
   `(ni,nj)`. Costs a transpose on every encode/decode. Rejected for the
   default.

**Decision: option 2 (reverse shape+strides), and document it loudly.**
Consequences, which **must** be stated in `fortran-api.md` and the
example headers:

- Fortran→Tensogram→Fortran round-trips are **bit-identical**.
- A NumPy / C / xarray / Zarr reader sees an array of shape `[nj, ni]` —
  the **transpose** of the Fortran array. This is the single most common
  source of silent cross-language bugs; the parity test (§5.8, §8) exists
  to keep it honest.

This is **decided** (§10): physically transposing so all languages share
the same logical shape (option 3) is explicitly *not* the default.

### 5.2 Native `real(:,:)` with zero C/Rust changes

Because tensor data crosses the boundary as **raw bytes + shape-as-JSON
text** (§3), a Fortran array *descriptor* never needs to reach C. All
array ergonomics live in the Fortran wrapper, binding directly to the
existing `libtensogram`.

Encode mechanism:

1. Take a **`contiguous`** assumed-shape dummy: `real(c_float), target,
   contiguous, intent(in) :: a(:,:)`.
2. Read `shape(a)`; emit the **reversed** extents into the descriptor
   JSON (§5.1).
3. `c_loc(a)` for the buffer pointer; `c_sizeof(a)` (or
   `size(a)*storage_size(a)/8`) for the byte length.

The linchpin is the **`CONTIGUOUS` attribute (F2008)**: if the caller
passes a non-contiguous slice (`a(1:n:2, :)`), the compiler silently
gathers it into a contiguous temporary for the call (copy-in), so
`c_loc` is **always** valid and the wrapper never walks strides. Since
the wire format is contiguous bytes, that gather is unavoidable work the
language does for us.

Decode mirror: `c_f_pointer(tgm_object_data(...), fptr, [reversed
extents])` for a zero-copy view tied to the `tgm_message_t` handle's
lifetime, or copy into a user `allocatable :: out(:,:)` to decouple from
the handle.

Genericity over rank and kind stays Fortran-side: either rank/kind
specific module procedures behind a **generic interface**, or a single
assumed-rank `dimension(..)` procedure resolved with `select rank`
(F2018), keeping `c_loc` strictly inside Fortran.

`CFI_cdesc_t` (passing the descriptor itself to C) is only worth it for
zero-copy non-contiguous input — deferred (§4.1, Path C).

### 5.3 Strings and JSON metadata

- Generic marshalling helpers: `character(len=*)` → NUL-terminated
  `c_char` array (allocatable + `target` so `c_loc` is valid); returned
  `const char*` → Fortran `allocatable` string via `strlen` +
  `c_f_pointer`.
- **Building the JSON — zero-dependency (DECIDED, §10).** A hand-built
  descriptor builder handles the fixed descriptor fields; *structured
  application metadata* (`base[i]`, namespaces) is assembled by a small
  internal key/value/string escaper in the Fortran layer — **no
  `json-fortran` dependency**, so the binding depends only on
  `libtensogram`. The escaper must cover the JSON string-escape set
  (`"`, `\`, control characters) to avoid emitting malformed metadata.
- The descriptor JSON must match the documented schema
  (`docs/src/guide/c-api.md` quick start): `type`, `ndim`, `shape`,
  `strides`, `dtype`, `byte_order`, `encoding`, `filter`, `compression`.
  Emit `ndim`, `strides`, and `byte_order` explicitly (the reference
  scaffold currently omits them — see §11).

### 5.4 Handles and lifetimes

- Wrap each `tgm_*_t*` in a derived type holding a `type(c_ptr)`, with a
  `final` procedure calling the matching `*_free`/`*_close`, plus an
  **idempotent explicit** `free`/`close` binding (guard with
  `c_associated`, null the pointer after release → double-free
  impossible on a single object).
- Provide explicit `close`/`free` methods too: Fortran finalizers do
  **not** fire on every path (e.g. `error stop`) and interact awkwardly
  with copy semantics.
- **Copy-safety — DECIDED: non-copyable handles (approach "A+", §10).**
  Fortran has no way to *delete* the copy: `b = a` shallow-copies the
  `c_ptr`, so two objects alias one Rust resource and both finalisers
  free it → double-free. The idempotent guard above only stops the
  *same-object* double-free, not this *aliased* one. We therefore:
  1. **design the API so handles are never implicitly copied** — factory
     procedures return via `intent(out)` arguments (never function
     results); handles are passed `intent(in)` / `intent(inout)` by
     reference; never stored in value arrays;
  2. **define `assignment(=)` to `error stop`** with a clear message, so
     any accidental `b = a` (or other whole-type copy) aborts **loudly at
     the copy site** instead of corrupting the heap later — Fortran can't
     prevent the copy, but it can make it fail deterministically;
  3. keep the idempotent explicit `free` / `close` + the `final`
     procedure as the normal path and backstop.
  **Reference counting is rejected** (see §10): it is the most code, its
  correctness rides on the two shakiest Fortran mechanisms
  (defined-assignment-on-implicit-copies and finalisers), and a
  thread-safe count would need atomics because `libtensogram` is used
  multi-threaded (free-threaded Python, OpenMP in IFS). A test must
  assert the `error stop` guard does **not** fire on the legitimate
  `intent(out)` factory / component-assignment paths.

### 5.5 Error mapping (and a single-source-of-truth check)

- Mirror `tgm_error` as Fortran `integer(c_int)` parameters using the
  **exact C names** (`TGM_ERROR_OK`, `TGM_ERROR_FRAMING`, …) so the
  mirror is obviously a mirror.
- Provide a checker: no-op on `TGM_ERROR_OK`; otherwise pull
  `tgm_last_error()` (fallback to `tgm_error_string`) and either return
  it via an optional `ierr`/message or `error stop`. Use the idiomatic
  optional trailing `ierr` argument; do not fake exceptions.
- **AGENTS.md compliance**: "error code lists" are explicitly called out
  as a value that must have one source of truth with a consistency
  check. Add a CI test that asserts the Fortran `TGM_ERROR_*` parameters
  match `tensogram.h` (parse the header, or assert the known 0–13
  mapping and fail if the header gains a variant the mirror lacks).

### 5.6 Build, packaging, and the "is it header-only?" question

**There is no header-only tier in Fortran.** A C++ header-only wrapper
works because bodies inline into each translation unit. A Fortran `.mod`
is interface/metadata only — it cannot carry procedure *bodies*. So:

- The *only* "zero compiled code of our own" Fortran binding is one made
  purely of `interface` blocks binding straight to `tgm_*` symbols. That
  hands users the raw C-style API (hand-built JSON, manual `c_loc`,
  manual free) — i.e. **not** the `real(:,:)` ergonomics we want.
- The moment we want `real(:,:)`, string marshalling, finalizers, or
  `ierr` mapping, we have procedure *bodies* → the binding **is** a small
  compiled Fortran library (`libtensogram_f.a` + `.mod` files).

Crucially: that compiled artifact links against the **existing**
`libtensogram` and needs **no new C or Rust piece**. Adding `real(:,:)`
moves us from "zero compiled code" to "one small Fortran library" — *not*
to "Fortran + a C shim + Rust changes." This is exactly the
`eccodes_f90` model; it is the Fortran norm, not a regression.

Packaging (**DECIDED: CMake primary, fpm retained — §10**):

- Link via the shipped `tensogram.pc` (`pkg-config --cflags --libs
  tensogram`).
- **CMake is the system of record.** It discovers the FFI natively
  (`find_package(PkgConfig)` → `pkg_check_modules(... IMPORTED_TARGET
  tensogram)` → `PkgConfig::TENSOGRAM`), builds and installs the library
  + `.mod` files, and integrates with the wider package (a superbuild can
  `add_subdirectory(fortran)` once the FFI is installed). CMake is the
  more widely used compiled-language tool and the natural fit for
  IFS/HPC sites.
- **fpm is retained** as a thin Fortran-native devloop. Its one friction
  is that fpm has **no native pkg-config integration**, so the external
  `libtensogram` is wired in by injecting flags at invocation —
  `fpm test --flag "$(pkg-config --cflags tensogram)" --link-flag
  "$(pkg-config --libs tensogram)"`, hidden behind a `make
  fortran-fpm-test` wrapper. `fpm.toml` keeps `[build] link =
  ["tensogram"]` and auto-globs `fortran/src/`.
- **Anti-drift**: the two build descriptions share the same sources (fpm
  globs the source dir; CMake lists files explicitly), so CI **builds the
  binding both ways** to catch any divergence. Keeping the binding to
  one/few module files makes this trivial.
- Ensure `.mod` files install somewhere discoverable.
- Live in `fortran/` beside `cpp/`; examples in `examples/fortran/`.

### 5.7 Async — deferred

The async surface uses C **function-pointer completion callbacks** and
opaque task handles. Fortran *can* supply a `bind(C)` callback, but it's
the hardest part and most NWP code is content with the blocking
`tgm_async_task_join_*` variants. Ship synchronous-only first; treat
async as a later, demand-driven phase (if ever). Even when tackled, bind
the blocking join variants before callbacks.

### 5.8 Pre-1.0 ABI churn → parity testing

Tensogram is pre-1.0 (0.21, the C ABI may change between minor releases,
SONAME bumps every minor). Mitigations baked into this plan:

- **Pin** the binding to exact tensogram versions; add `fortran/fpm.toml`
  to the VERSION single-source-of-truth list (§9).
- **Fortran↔Python round-trip parity test in CI**: encode the same data
  in Fortran, decode in Python (and vice versa), assert array + metadata
  equality. This is what actually keeps a moving ABI from silently
  breaking the binding, and it doubles as the guard for the column-major
  contract (§5.1).
- **Error-enum consistency check** against `tensogram.h` (§5.5).

## 8. Verification & validation strategy (overall)

How we know it is correct, beyond per-milestone tests:

1. **Round-trip bit-identity** is the primary correctness gate for the
   lossless path (`encoding=filter=compression="none"`):
   `maxval(abs(out - in)) == 0`.
2. **Cross-language parity (the safety net)**: a Fortran writer + Python
   reader (and the reverse) asserting array **and** metadata equality —
   reusing the spirit of `tests/remote-parity/`. Guards both ABI drift
   and the column-major contract. Pin to the exact tensogram version.
3. **Compiler**: gfortran only for the initial arc (§10); `ifx` /
   `nvfortran` are deferred. The code stays standard `-std=f2018`, so
   only the *tested* matrix is narrowed, not the targeted language level.
   Treat warnings as errors (`-Wall -Wextra`).
4. **Memory safety across the FFI**: run the example/tests under
   `-fsanitize=address` (gfortran) and/or valgrind to catch double-free
   / leaks — especially the move-like handle caveat (§5.4) and the
   by-value `tgm_bytes_free`.
5. **Error-enum consistency** vs. `tensogram.h` (§5.5) in CI.
6. **`make all` clean** before any commit (AGENTS.md hard rule), with the
   Fortran targets wired in.

## 10. Decisions

All settled at planning time. (Numbered for cross-references elsewhere in
this document.)

1. **Column-major contract (§5.1) — DECIDED: reverse shape+strides at the
   boundary.** A Fortran `a(ni,nj)` is written with the on-wire
   descriptor shape/strides reversed to C order: Fortran↔Fortran
   round-trips are bit-identical, and a NumPy / C / xarray / Zarr reader
   sees the transpose `[nj, ni]`. Zero data movement; matches
   `docs/src/concepts/objects.md`. Must be documented loudly in
   `fortran-api.md`, the example headers, and guarded by the parity test
   (§5.8, §8). Physically transposing so all languages share the same
   logical shape (option 3) is explicitly *not* the default.
2. **Decode `verify_hash` default — DECIDED: off.** Match the library
   default (`DESIGN.md`): hash verification is opt-in. The binding still
   exposes the flag so callers can request it. The reference scaffold,
   which defaults it on, is corrected accordingly (§11.5).
3. **Compiler matrix — DECIDED: gfortran only for the initial arc.**
   Intel `ifx` and NVIDIA `nvfortran` are explicitly out of scope for now
   (revisit on demand). The binding still targets standard F2008/F2018
   (`-std=f2018`), so portability is preserved — only the *tested* matrix
   is gfortran.

4. **Metadata JSON — DECIDED: zero-dependency (hand-built).** No
   `json-fortran`; the binding's only dependency stays `libtensogram`.
   The descriptor JSON is small and fixed; structured application
   metadata (`base[i]`, namespaces) is assembled via a small internal
   key/value/string escaper in the Fortran layer to avoid quoting bugs.
   (§5.3.)
5. **Build system of record — DECIDED: CMake primary, fpm retained.**
   CMake discovers the FFI via pkg-config and integrates with the wider
   package; fpm stays as a Fortran-native devloop with pkg-config flags
   injected at invocation. CI builds both ways to prevent drift.
   (§5.6, §9.)
6. **Public naming — DECIDED: `tensogram_*`.** Public procedures and
   derived types use the `tensogram_*` namespace (`tensogram_message`,
   `tensogram_buffer`, …); the private raw `bind(C)` layer keeps the C
   `tgm_*` names. (Scaffold rename noted in §11.)
7. **Handle copy-safety — DECIDED: non-copyable handles ("A+").**
   Idempotent explicit `free` / `close` + a `final` backstop; an API that
   never implicitly copies handles (`intent(out)` factories, by-reference
   passing); and a defined `assignment(=)` that `error stop`s so any
   accidental `b = a` fails loudly at the copy site. Reference counting
   is rejected — most code, depends on the shakiest Fortran mechanisms
   (defined-assignment-on-implicit-copies, finalisers), and would need
   atomics under multi-threaded use. (§5.4.)

No open questions remain at planning time; new ones are raised against
the relevant milestone as implementation surfaces them.

## 11. Corrections to apply to the reference scaffold

Found by reading the actual 0.21.0 `tensogram.h`. The scaffold in §12 is
**compile- and link-verified as provided** (under `-Wall -Wextra
-std=f2018`, against a C stub), so it is reproduced verbatim; apply these
deliberately and re-verify when integrating:

1. **Error constant names.** The C header uses `TGM_ERROR_*`
   (`TGM_ERROR_OK`, `TGM_ERROR_FRAMING`, …, `TGM_ERROR_CANCELLED = 13`).
   The scaffold's Fortran parameters use `TGM_ERR_*`. **Rename to
   `TGM_ERROR_*`** to mirror the header exactly (single source of truth)
   and make the consistency check trivial. The numeric values (0–13)
   already match.
2. **Descriptor JSON completeness.** The documented schema
   (`docs/src/guide/c-api.md`) includes `ndim`, `shape`, `strides`,
   `dtype`, `byte_order`, `encoding`, `filter`, `compression`. The
   scaffold's `descriptor_json_r2_f32` emits only `shape`, `dtype`, and
   the pipeline fields. **Add `ndim`, explicit `strides` (`[ni, 1]` over
   the reversed shape `[nj, ni]`), and `byte_order` (native)** for a
   fully self-describing, schema-matching descriptor rather than relying
   on library-side defaults.
3. **Shape/ndim signedness.** `tgm_object_shape` / `tgm_object_ndim`
   return `uint64_t`; the scaffold maps them to `integer(c_int64_t)`.
   Bit-compatible for realistic sizes; keep, but note the signedness (no
   real tensor exceeds `i64::MAX` extents).
4. **`fpm.toml` version.** The scaffold says `version = "0.1.0"`; it must
   equal the repo `VERSION` (`0.21.0`) and be added to the VERSION-sync
   list (§9).
5. **Decode `verify_hash` default.** Decided **off** (§10) to match the
   library. Change the scaffold's `tensogram_decode` default from
   `verify_hash = .true.` to `.false.` (keep the flag exposed).
6. **Public type names + copy guard.** Per the `tensogram_*` naming
   decision (§10), rename the scaffold's handle types `tgm_buffer` →
   `tensogram_buffer` and `tgm_message` → `tensogram_message` (keep the
   `tgm_*` prefix only for the private `bind(C)` layer), and add a defined
   `assignment(=)` that `error stop`s to each handle type to make them
   non-copyable ("A+", §5.4).

### `fortran/src/tensogram.f90`

```fortran
!> tensogram — Fortran 2008/2018 interface to the Tensogram C ABI (libtensogram).
!>
!> SCAFFOLD. Covers the synchronous encode -> bytes -> decode -> array round-trip
!> for rank-2 float32 tensors. The private `bind(C)` interface block mirrors a
!> subset of `tensogram.h`; the public procedures add idiomatic Fortran strings,
!> ownership (finalizers), and the memory-order contract. Extend the
!> rank/dtype-specific wrappers and the interface block to cover more of the C
!> surface (file API, streaming, scan/iterators, validation, async ...).
!>
!> MEMORY-ORDER CONTRACT
!>   A Fortran array `a(ni, nj)` is written with the descriptor shape REVERSED to
!>   [nj, ni] (C/row-major order on the wire). Consequences:
!>     * Round-trips Fortran <-> Fortran are bit-identical (see the example).
!>     * A NumPy / C reader sees an array of shape (nj, ni) — the transpose.
!>   This mirrors docs/src/concepts/objects.md ("a Fortran-contiguous array has
!>   the strides reversed"). Decide once, document loudly; it is the single most
!>   common source of cross-language confusion.
!>
!> OWNERSHIP / LIFETIMES
!>   `tgm_buffer`  owns Rust-allocated encoded bytes (freed via tgm_bytes_free).
!>   `tgm_message` owns a decoded message handle  (freed via tgm_message_free).
!>   Both free in a `final` procedure and via an idempotent `free` binding.
!>   These types are MOVE-like: do not copy them by intrinsic assignment
!>   (`b = a`), or both copies will reference — and double-free — the same
!>   resource. Pass them by reference; let scope-exit finalization reclaim them.
!>
!> Build (against an installed tensogram.pc):
!>   gfortran -O2 fortran/src/tensogram.f90 examples/fortran/encode_decode.f90 \
!>            $(pkg-config --cflags --libs tensogram) -o encode_decode
module tensogram
   use, intrinsic :: iso_c_binding
   use, intrinsic :: iso_fortran_env, only : error_unit
   implicit none
   private

   ! ---- Public API ---------------------------------------------------------
   public :: tgm_buffer, tgm_message
   public :: tensogram_encode_r2_f32
   public :: tensogram_decode
   public :: tensogram_num_objects, tensogram_object_ndim
   public :: tensogram_object_shape, tensogram_object_dtype
   public :: tensogram_object_to_r2_f32
   public :: tensogram_strerror, tensogram_last_error, tensogram_check
   public :: TGM_OK, TGM_ERR_FRAMING, TGM_ERR_METADATA, TGM_ERR_ENCODING,     &
             TGM_ERR_COMPRESSION, TGM_ERR_OBJECT, TGM_ERR_IO,                  &
             TGM_ERR_HASH_MISMATCH, TGM_ERR_INVALID_ARG, TGM_ERR_END_OF_ITER, &
             TGM_ERR_REMOTE, TGM_ERR_MISSING_HASH, TGM_ERR_TIMEOUT,           &
             TGM_ERR_CANCELLED

   ! ---- tgm_error enum (must match tensogram.h) ----------------------------
   integer(c_int), parameter :: TGM_OK              = 0
   integer(c_int), parameter :: TGM_ERR_FRAMING     = 1
   integer(c_int), parameter :: TGM_ERR_METADATA    = 2
   integer(c_int), parameter :: TGM_ERR_ENCODING    = 3
   integer(c_int), parameter :: TGM_ERR_COMPRESSION = 4
   integer(c_int), parameter :: TGM_ERR_OBJECT      = 5
   integer(c_int), parameter :: TGM_ERR_IO          = 6
   integer(c_int), parameter :: TGM_ERR_HASH_MISMATCH = 7
   integer(c_int), parameter :: TGM_ERR_INVALID_ARG   = 8
   integer(c_int), parameter :: TGM_ERR_END_OF_ITER   = 9
   integer(c_int), parameter :: TGM_ERR_REMOTE        = 10
   integer(c_int), parameter :: TGM_ERR_MISSING_HASH  = 11
   integer(c_int), parameter :: TGM_ERR_TIMEOUT       = 12
   integer(c_int), parameter :: TGM_ERR_CANCELLED     = 13

   ! ---- Interoperable POD struct: tgm_bytes_t ------------------------------
   !  typedef struct { uint8_t *data; size_t len; } tgm_bytes_t;
   type, bind(C) :: tgm_bytes_t
      type(c_ptr)       :: data = c_null_ptr
      integer(c_size_t) :: len  = 0_c_size_t
   end type tgm_bytes_t

   ! ---- Owned encoded buffer (RAII over tgm_bytes_t) -----------------------
   type :: tgm_buffer
      type(tgm_bytes_t) :: raw
   contains
      procedure :: as_array => buffer_as_array   !> copy bytes into int8(:)
      procedure :: size     => buffer_size       !> length in bytes
      procedure :: free     => buffer_free        !> idempotent release
      final     :: buffer_final
   end type tgm_buffer

   ! ---- Owned decoded message (RAII over tgm_message_t*) -------------------
   type :: tgm_message
      type(c_ptr) :: ptr = c_null_ptr
   contains
      procedure :: free => message_free            !> idempotent release
      final     :: message_final
   end type tgm_message

   ! =========================================================================
   !  Raw C ABI — a subset of tensogram.h.
   !  Convention: every input pointer (const T*) is passed as `type(c_ptr),
   !  value`; out-handles (T**) as `type(c_ptr)` by reference; returned const
   !  char* / const T* as `type(c_ptr)`. cbindgen emits `tgm_error` as a C
   !  `int`, hence `integer(c_int)`.
   ! =========================================================================
   interface
      !> const char *tgm_last_error(void);
      function c_tgm_last_error() bind(C, name="tgm_last_error") result(p)
         import :: c_ptr
         type(c_ptr) :: p
      end function

      !> const char *tgm_error_string(tgm_error err);
      function c_tgm_error_string(err) bind(C, name="tgm_error_string") result(p)
         import :: c_ptr, c_int
         integer(c_int), value :: err
         type(c_ptr) :: p
      end function

      !> void tgm_bytes_free(tgm_bytes_t buf);   [struct passed BY VALUE]
      subroutine c_tgm_bytes_free(buf) bind(C, name="tgm_bytes_free")
         import :: tgm_bytes_t
         type(tgm_bytes_t), value :: buf
      end subroutine

      !> tgm_error tgm_encode(const char *metadata_json,
      !>                      const uint8_t *const *data_ptrs,
      !>                      const size_t *data_lens, size_t num_objects,
      !>                      const char *hash_algo, uint32_t threads,
      !>                      tgm_bytes_t *out);
      function c_tgm_encode(meta, ptrs, lens, n, hash, threads, out) &
            bind(C, name="tgm_encode") result(err)
         import :: c_ptr, c_size_t, c_int32_t, c_int, tgm_bytes_t
         type(c_ptr),        value       :: meta
         type(c_ptr),        value       :: ptrs
         type(c_ptr),        value       :: lens
         integer(c_size_t),  value       :: n
         type(c_ptr),        value       :: hash
         integer(c_int32_t), value       :: threads
         type(tgm_bytes_t),  intent(out) :: out
         integer(c_int)                  :: err
      end function

      !> tgm_error tgm_decode(const uint8_t *buf, size_t buf_len,
      !>                      int32_t native_byte_order, uint32_t threads,
      !>                      int32_t verify_hash, tgm_message_t **out);
      function c_tgm_decode(buf, buf_len, native_bo, threads, verify, out) &
            bind(C, name="tgm_decode") result(err)
         import :: c_ptr, c_size_t, c_int32_t, c_int
         type(c_ptr),        value       :: buf
         integer(c_size_t),  value       :: buf_len
         integer(c_int32_t), value       :: native_bo
         integer(c_int32_t), value       :: threads
         integer(c_int32_t), value       :: verify
         type(c_ptr),        intent(out) :: out
         integer(c_int)                  :: err
      end function

      !> size_t tgm_message_num_objects(const tgm_message_t *msg);
      function c_tgm_message_num_objects(msg) &
            bind(C, name="tgm_message_num_objects") result(n)
         import :: c_ptr, c_size_t
         type(c_ptr), value :: msg
         integer(c_size_t)  :: n
      end function

      !> uint64_t tgm_object_ndim(const tgm_message_t *msg, size_t index);
      function c_tgm_object_ndim(msg, idx) bind(C, name="tgm_object_ndim") result(n)
         import :: c_ptr, c_size_t, c_int64_t
         type(c_ptr),       value :: msg
         integer(c_size_t), value :: idx
         integer(c_int64_t)       :: n
      end function

      !> const uint64_t *tgm_object_shape(const tgm_message_t *msg, size_t index);
      function c_tgm_object_shape(msg, idx) bind(C, name="tgm_object_shape") result(p)
         import :: c_ptr, c_size_t
         type(c_ptr),       value :: msg
         integer(c_size_t), value :: idx
         type(c_ptr)              :: p
      end function

      !> const char *tgm_object_dtype(const tgm_message_t *msg, size_t index);
      function c_tgm_object_dtype(msg, idx) bind(C, name="tgm_object_dtype") result(p)
         import :: c_ptr, c_size_t
         type(c_ptr),       value :: msg
         integer(c_size_t), value :: idx
         type(c_ptr)              :: p
      end function

      !> const uint8_t *tgm_object_data(const tgm_message_t *msg,
      !>                               size_t decoded_index, size_t *out_len);
      function c_tgm_object_data(msg, idx, out_len) &
            bind(C, name="tgm_object_data") result(p)
         import :: c_ptr, c_size_t
         type(c_ptr),       value       :: msg
         integer(c_size_t), value       :: idx
         integer(c_size_t), intent(out) :: out_len
         type(c_ptr)                    :: p

      !> void tgm_message_free(tgm_message_t *msg);

         import :: c_ptr

      function c_strlen(s) bind(C, name="strlen") result(n)
         import :: c_ptr, c_size_t
         type(c_ptr), value :: s
         integer(c_size_t)  :: n
      end function

contains

   ! =========================================================================
   !  String helpers
   ! =========================================================================

   !> Fortran string -> NUL-terminated c_char array. `c` is allocatable+target
   !> so the caller can take c_loc(c) and pass it to a `const char*` parameter.
   subroutine f_to_cstr(f, c)
      character(len=*),                    intent(in)  :: f
      character(kind=c_char), allocatable, intent(out) :: c(:)
      integer :: i, n
      n = len(f)
      allocate(c(n + 1))
      do i = 1, n
         c(i) = f(i:i)
      end do
      c(n + 1) = c_null_char
   end subroutine f_to_cstr

   !> C `const char*` -> Fortran allocatable string. Returns '' for NULL.
   function cptr_to_fstr(p) result(f)
      type(c_ptr), intent(in)         :: p
      character(len=:), allocatable   :: f
      character(kind=c_char), pointer :: buf(:)
      integer(c_size_t) :: n
      integer :: i
      if (.not. c_associated(p)) then
         f = ''
         return
      end if
      n = c_strlen(p)
      call c_f_pointer(p, buf, [n])
      allocate(character(len=int(n)) :: f)
      do i = 1, int(n)
         f(i:i) = buf(i)
      end do
   end function cptr_to_fstr

   !> Integer -> compact decimal string (no leading blanks). Used by the JSON
   !> descriptor builder. For structured application metadata, prefer a real
   !> JSON library (e.g. json-fortran) rather than hand-built strings.
   pure function itoa(i) result(s)
      integer, intent(in)           :: i
      character(len=:), allocatable :: s
      character(len=32)             :: tmp
      write (tmp, '(i0)') i
      s = trim(tmp)
   end function itoa

   !> Build the metadata JSON for one rank-2 float32 object. The shape is
   !> REVERSED (nj, ni) to honour the column-major contract. `extra`, if given,
   !> is a raw JSON fragment of additional TOP-LEVEL keys appended verbatim,
   !> e.g. '"base":[{"product":{"name":"t","units":"K"}}]' — advanced use.
   function descriptor_json_r2_f32(ni, nj, extra) result(js)
      integer,          intent(in)           :: ni, nj
      character(len=*), intent(in), optional :: extra
      character(len=:), allocatable          :: js
      js = '{"descriptors":[{"type":"ntensor","shape":[' //          &
           itoa(nj) // ',' // itoa(ni) // '],"dtype":"float32",' //  &
           '"encoding":"none","filter":"none","compression":"none"}]'
      if (present(extra)) then
         if (len_trim(extra) > 0) js = js // ',' // trim(extra)
      end if
      js = js // '}'
   end function descriptor_json_r2_f32

   ! =========================================================================
   !  Error helpers
   ! =========================================================================

   !> Static description of an error code (from tgm_error_string).
   function tensogram_strerror(err) result(msg)
      integer(c_int), intent(in)    :: err
      character(len=:), allocatable :: msg
      msg = cptr_to_fstr(c_tgm_error_string(err))
   end function tensogram_strerror

   !> The thread-local message left by the most recent failing FFI call.
   function tensogram_last_error() result(msg)
      character(len=:), allocatable :: msg
      msg = cptr_to_fstr(c_tgm_last_error())
   end function tensogram_last_error

   !> Idiomatic guard: no-op on TGM_OK, otherwise print details and stop.
   !> Use the optional `context` to say which call failed.
   subroutine tensogram_check(err, context)
      integer(c_int),   intent(in)           :: err
      character(len=*), intent(in), optional :: context
      character(len=:), allocatable :: detail
      if (err == TGM_OK) return
      detail = tensogram_last_error()
      if (len(detail) == 0) detail = tensogram_strerror(err)
      if (present(context)) then
         write (error_unit, '(a)') 'tensogram: ' // trim(context) // ': ' // detail
      else
         write (error_unit, '(a)') 'tensogram: ' // detail
      end if
      error stop 1
   end subroutine tensogram_check

   ! =========================================================================
   !  tgm_buffer methods (encoded bytes owned by Rust)
   ! =========================================================================

   function buffer_size(self) result(n)
      class(tgm_buffer), intent(in) :: self
      integer(c_size_t) :: n
      n = self%raw%len
   end function buffer_size

   !> Copy the owned bytes into a fresh Fortran int8 array. The copy decouples
   !> the data from the buffer's lifetime, so it is safe to free() afterwards.
   subroutine buffer_as_array(self, out)
      class(tgm_buffer), intent(in)                :: self
      integer(c_int8_t), allocatable, intent(out)  :: out(:)
      integer(c_int8_t), pointer :: view(:)
      if (.not. c_associated(self%raw%data) .or. self%raw%len == 0_c_size_t) then
         allocate(out(0))
         return
      end if
      call c_f_pointer(self%raw%data, view, [self%raw%len])
      allocate(out(self%raw%len))
      out = view
   end subroutine buffer_as_array

   subroutine buffer_free(self)
      class(tgm_buffer), intent(inout) :: self
      if (c_associated(self%raw%data)) then
         call c_tgm_bytes_free(self%raw)       ! struct passed by value
         self%raw%data = c_null_ptr
         self%raw%len  = 0_c_size_t
      end if
   end subroutine buffer_free

   subroutine buffer_final(self)
      type(tgm_buffer), intent(inout) :: self
      call self%free()
   end subroutine buffer_final

   ! =========================================================================
   !  tgm_message methods (decoded handle owned by Rust)
   ! =========================================================================

   subroutine message_free(self)
      class(tgm_message), intent(inout) :: self
      if (c_associated(self%ptr)) then
         call c_tgm_message_free(self%ptr)
         self%ptr = c_null_ptr
      end if
   end subroutine message_free

   subroutine message_final(self)
      type(tgm_message), intent(inout) :: self
      call self%free()
   end subroutine message_final

   ! =========================================================================
   !  Public encode / decode / inspect
   ! =========================================================================

   !> Encode a single rank-2 float32 tensor as a one-object Tensogram message.
   !> `a` may be a non-contiguous slice: the `contiguous` attribute makes the
   !> compiler gather it into a temporary, so c_loc(a) is always valid.
   subroutine tensogram_encode_r2_f32(a, buf, err, metadata_json, hash)
      real(c_float), target, contiguous, intent(in)  :: a(:,:)
      type(tgm_buffer),                  intent(out)  :: buf
      integer(c_int),                    intent(out)  :: err
      character(len=*), intent(in), optional :: metadata_json  ! raw extra top-level JSON keys
      character(len=*), intent(in), optional :: hash           ! "xxh3" (default); "" => none

      character(kind=c_char), allocatable, target :: meta_c(:), hash_c(:)
      character(len=:),       allocatable         :: meta_s, hash_s
      type(c_ptr),       target :: ptrs(1)
      integer(c_size_t), target :: lens(1)
      type(c_ptr)               :: hash_ptr

      meta_s = descriptor_json_r2_f32(size(a, 1), size(a, 2), metadata_json)
      call f_to_cstr(meta_s, meta_c)

      if (present(hash)) then
         hash_s = hash
      else
         hash_s = 'xxh3'
      end if
      if (len(hash_s) == 0) then
         hash_ptr = c_null_ptr                     ! NULL hash_algo => no hash
      else
         call f_to_cstr(hash_s, hash_c)
         hash_ptr = c_loc(hash_c)
      end if

      ptrs(1) = c_loc(a)                            ! contiguous + target => valid
      lens(1) = int(size(a, kind=c_size_t), c_size_t) &
                * int(storage_size(a) / 8, c_size_t)   ! total bytes (= ni*nj*4)

      err = c_tgm_encode(c_loc(meta_c), c_loc(ptrs), c_loc(lens), &
                         1_c_size_t, hash_ptr, 0_c_int32_t, buf%raw)
   end subroutine tensogram_encode_r2_f32

   !> Decode a wire-format message (the bytes produced by encode) into a handle.
   subroutine tensogram_decode(wire, msg, err, verify_hash, native_byte_order)
      integer(c_int8_t), target, contiguous, intent(in)  :: wire(:)
      type(tgm_message),                     intent(out) :: msg
      integer(c_int),                        intent(out) :: err
      logical, intent(in), optional :: verify_hash        ! default .true.
      logical, intent(in), optional :: native_byte_order  ! default .true. (host endianness)
      integer(c_int32_t) :: vh, nbo
      type(c_ptr)        :: out

      vh = 1_c_int32_t
      if (present(verify_hash)) vh = merge(1_c_int32_t, 0_c_int32_t, verify_hash)
      nbo = 1_c_int32_t
      if (present(native_byte_order)) &
         nbo = merge(1_c_int32_t, 0_c_int32_t, native_byte_order)

      err = c_tgm_decode(c_loc(wire), size(wire, kind=c_size_t), &
                         nbo, 0_c_int32_t, vh, out)
      if (err == TGM_OK) msg%ptr = out
   end subroutine tensogram_decode

   !> Number of objects in the decoded message.
   function tensogram_num_objects(msg) result(n)
      type(tgm_message), intent(in) :: msg
      integer :: n
      n = int(c_tgm_message_num_objects(msg%ptr))
   end function tensogram_num_objects

   !> Rank of object `iobj` (1-based).
   function tensogram_object_ndim(msg, iobj) result(nd)
      type(tgm_message), intent(in) :: msg
      integer,           intent(in) :: iobj
      integer :: nd
      nd = int(c_tgm_object_ndim(msg%ptr, int(iobj - 1, c_size_t)))
   end function tensogram_object_ndim

   !> Object extents in FORTRAN (column-major) order — the on-wire descriptor
   !> shape REVERSED. ext(1) is the fastest-varying axis.
   function tensogram_object_shape(msg, iobj) result(ext)
      type(tgm_message), intent(in)   :: msg
      integer,           intent(in)   :: iobj
      integer(c_int64_t), allocatable :: ext(:)
      integer(c_int64_t), pointer     :: cshape(:)
      integer     :: nd, k
      type(c_ptr) :: p
      nd = tensogram_object_ndim(msg, iobj)
      p  = c_tgm_object_shape(msg%ptr, int(iobj - 1, c_size_t))
      call c_f_pointer(p, cshape, [nd])
      allocate(ext(nd))
      do k = 1, nd
         ext(k) = cshape(nd - k + 1)              ! reverse -> Fortran order
      end do
   end function tensogram_object_shape

   !> dtype string of object `iobj` (e.g. "float32").
   function tensogram_object_dtype(msg, iobj) result(dt)
      type(tgm_message), intent(in) :: msg
      integer,           intent(in) :: iobj
      character(len=:), allocatable :: dt
      dt = cptr_to_fstr(c_tgm_object_dtype(msg%ptr, int(iobj - 1, c_size_t)))
   end function tensogram_object_dtype

   !> Copy decoded object `iobj` (must be rank-2 float32) into a Fortran array
   !> shaped (ni, nj), matching what tensogram_encode_r2_f32 wrote. The copy is
   !> deliberate: the raw pointer aliases message-owned memory and would dangle
   !> once `msg` is freed. (For zero-copy, expose the `view` pointer instead and
   !> keep `msg` alive — see the inline note.)
   subroutine tensogram_object_to_r2_f32(msg, iobj, out, err)
      type(tgm_message),          intent(in)  :: msg
      integer,                    intent(in)  :: iobj
      real(c_float), allocatable, intent(out) :: out(:,:)
      integer(c_int),             intent(out) :: err
      integer(c_int64_t), allocatable :: ext(:)
      character(len=:),   allocatable :: dt
      real(c_float),      pointer     :: view(:,:)
      integer(c_size_t) :: nbytes
      type(c_ptr)       :: dptr

      err = TGM_OK
      dt = tensogram_object_dtype(msg, iobj)
      if (dt /= 'float32') then
         err = TGM_ERR_OBJECT
         return
      end if
      ext = tensogram_object_shape(msg, iobj)      ! Fortran (reversed) order
      if (size(ext) /= 2) then
         err = TGM_ERR_OBJECT
         return
      end if

      dptr = c_tgm_object_data(msg%ptr, int(iobj - 1, c_size_t), nbytes)
      if (.not. c_associated(dptr)) then
         err = TGM_ERR_OBJECT
         return
      end if

      ! C-contiguous bytes of shape [nj,ni] == Fortran-contiguous [ni,nj].
      call c_f_pointer(dptr, view, [int(ext(1)), int(ext(2))])
      allocate(out(int(ext(1)), int(ext(2))))
      out = view                                   ! decouple from msg lifetime
   end subroutine tensogram_object_to_r2_f32

end module tensogram
```

### `examples/fortran/encode_decode.f90`

```fortran
!> examples/fortran/encode_decode.f90
!>
!> End-to-end Tensogram round-trip from Fortran, mirroring
!> examples/python/01_encode_decode.py:
!>
!>   field(ni,nj)  --encode-->  tgm_buffer (Rust-owned bytes)
!>                 --copy out-->  wire(:)  (Fortran int8 array)
!>                 --decode-->   tgm_message handle
!>                 --extract-->  out(ni,nj)
!>
!> and asserts the round-trip is bit-identical. Build against an installed
!> tensogram.pc:
!>
!>   gfortran -O2 fortran/src/tensogram.f90 examples/fortran/encode_decode.f90 \
!>            $(pkg-config --cflags --libs tensogram) -o encode_decode
!>   ./encode_decode
program encode_decode
   use, intrinsic :: iso_c_binding, only : c_float, c_int8_t, c_int
   use tensogram
   implicit none

   integer, parameter :: NI = 100, NJ = 200
   real(c_float)                  :: field(NI, NJ)
   real(c_float),    allocatable  :: out(:,:)
   integer(c_int8_t), allocatable :: wire(:)
   type(tgm_buffer)  :: buf
   type(tgm_message) :: msg
   integer(c_int)    :: err
   integer           :: i, j
   real(c_float)     :: maxdiff

   ! 1. Synthesise a field (column 100*i + 0.001*j so every element is distinct).
   do j = 1, NJ
      do i = 1, NI
         field(i, j) = real(i, c_float) + 0.001_c_float * real(j, c_float)
      end do
   end do
   print '(a,i0,a,i0,a)', 'Input:   shape=(', NI, ',', NJ, ')  dtype=float32'

   ! 2. Encode -> Rust-owned byte buffer (lossless: encoding/compression "none").
   call tensogram_encode_r2_f32(field, buf, err)
   call tensogram_check(err, 'encode')
   print '(a,i0,a)',      'Message: ', buf%size(), ' bytes'

   ! 3. tgm_bytes_t round-trip: copy the wire bytes out, then release the buffer.
   call buf%as_array(wire)
   call buf%free()                       ! optional: free Rust memory early

   ! 4. Decode the wire bytes -> message handle.
   call tensogram_decode(wire, msg, err)
   call tensogram_check(err, 'decode')
   print '(a,i0,a)',      'Decoded: ', tensogram_num_objects(msg), ' object(s)'
   print '(a,a)',         '  dtype = ', tensogram_object_dtype(msg, 1)

   ! 5. Extract object 1 back into a Fortran array shaped (ni, nj).
   call tensogram_object_to_r2_f32(msg, 1, out, err)
   call tensogram_check(err, 'object_to_r2_f32')
   print '(a,i0,a,i0,a)', '  shape = (', size(out, 1), ',', size(out, 2), ')'

   ! 6. Verify the Fortran <-> Fortran round-trip is identity.
   maxdiff = maxval(abs(out - field))
   print '(a,es12.5)',    '  max |out - field| = ', maxdiff
   if (maxdiff == 0.0_c_float) then
      print '(a)', 'Round-trip OK (bit-identical).'
   else
      print '(a)', 'Round-trip MISMATCH.'
      error stop 1
   end if

   ! buf already freed above; `msg` and `out` are finalised automatically at
   ! program termination (msg's handle is released via tgm_message_free).
end program encode_decode
```
